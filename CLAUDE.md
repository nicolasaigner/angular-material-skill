# CLAUDE.md — angular-material-skill

Public MIT project: an Agent Skill for **Angular Material** and the **Angular CDK**, generated
deterministically from [`angular/components`](https://github.com/angular/components). This file
gives an agent working *on this repository* the conventions to follow.

## What this repo is

- `skills/angular-material/` — the distributable skill (`SKILL.md` + `references/`). This is the
  product; it is what consumers install.
- `tools/` — the generation/sync pipeline that produces the references.

## Conventions

- **Runtime:** Node 22 (`.nvmrc`), managed with fnm/nvm; **npm** (never pnpm/yarn/bun).
- **No runtime dependencies:** the pipeline uses only Node's standard library (`node:fs/promises`,
  `node:crypto`, `node:path`, global `fetch`, `node:test`). Keep it that way (YAGNI).
- **ESM** (`.mjs`) everywhere.
- **Tests:** `npm test` = `node --test`. The pipeline is **deterministic** (same input → same
  byte output) and its network access is **injected** (every function that hits the network takes
  its getter as a parameter), so the suite runs fully offline.

## Pipeline (tools/)

`discover-sources.mjs` (reads the upstream tree via the GitHub trees API → `sources.generated.json`)
→ `sources.mjs` (reads that JSON) → `fetch-upstream.mjs` + `distill.mjs` (fetch prose + examples,
render each reference) → `sync.mjs` (diff by content hash + `manifest.json`) → `gen-index.mjs`
(the grouped `_index.md`). `check-release.mjs` is the release notifier;
`.github/workflows/sync-upstream.yml` automates the weekly check → regenerate → open-PR flow.

## Rules

- **Never hand-edit generated files** (`skills/angular-material/references/*.md`, `manifest.json`,
  `tools/lib/sources.generated.json`). They are regenerated by the pipeline; a manual edit is
  silently overwritten and hides a real pipeline bug. If a reference looks wrong, fix the
  corresponding library in `tools/lib/` and regenerate.
- **Scope rule for what becomes a reference:** a path `<c>/<c>.md` (dir === basename) becomes a
  reference — zero curation. CDK entries get a `cdk-` name prefix; global guides get
  `category: 'guide'` and no examples.
- Keep the pipeline dependency-free and deterministic; follow existing patterns.
